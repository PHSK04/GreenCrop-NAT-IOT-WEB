# โครงสร้างและหน้าที่ของไฟล์โค้ด (Project Code Structure)

เอกสารนี้รวบรวมคำอธิบายการทำงานของไฟล์ต่างๆ ในโปรเจกต์ เพื่อให้เข้าใจภาพรวมของระบบ

## 1. ไฟล์หลัก (Core Files)

- **`src/main.tsx`**: จุดเริ่มต้นของโปรแกรม (Entry Point) ทำหน้าที่นำ React App ไปแสดงผลบนหน้าเว็บ
- **`src/App.tsx`**: ตัวจัดการหน้าหลัก ทำหน้าที่ตรวจสอบสถานะการล็อกอิน (Login State)
  - ถ้าล็อกอินแล้ว -> แสดงหน้า `Dashboard`
  - ถ้ายังไม่ล็อกอิน -> แสดงหน้า `Login`
- **`src/index.css`**: ไฟล์ตกแต่งรูปแบบ (CSS) หลักของทั้งโปรเจกต์ รวมถึงกำหนด Font และ Tailwind CSS

## 2. ระบบจัดการข้อมูล (State Management)

- **`src/contexts/MachineContext.tsx`**: **หัวใจสำคัญของระบบ IoT**
  - จัดการการเชื่อมต่อ **MQTT** (รับ-ส่งข้อมูลกับฮาร์ดแวร์)
  - เก็บค่าสถานะของ Sensor ต่างๆ (pH, EC, ระดับน้ำ) ให้ทุกหน้าเรียกใช้ได้
  - มีระบบ **Simulation** (จำลองค่า) เมื่อไม่มีการเชื่อมต่อจริง เพื่อให้หน้าเว็บทำงานได้ไม่ error

## 3. ส่วนประกอบหลัก (Main Components)

- **`src/components/Dashboard.tsx`**: โครงหน้าต่างหลักของเว็บ
  - จัดการ **Side Menu** (แถบเมนูซ้ายมือ)
  - รองรับการเปลี่ยนภาษา (TH/EN)
  - รองรับ Responsive (ซ่อนเมนูเมื่อจอเล็ก)
  - ทำหน้าที่เลือก "หน้าย่อย" มาแสดงผลตรงกลาง
- **`src/components/Login.tsx`**: หน้าเข้าสู่ระบบ
  - มี Animation พื้นหลัง
  - ตรวจสอบ Email/Password (ปัจจุบันจำลองเป็น `admin@smartiot.com`)
- **`src/components/mode-toggle.tsx`**: ปุ่มสลับโหมดมืด/สว่าง (Dark/Light Mode)

## 4. หน้าจอการทำงาน (Pages)

ไฟล์ในโฟลเดอร์ `src/components/pages/` คือหน้าเนื้อหาแต่ละส่วน:

- **`DashboardPage.tsx`**: **หน้าแรก (Main Dashboard)**
  - แสดงภาพจำลองระบบ (Visualizer)
  - แสดงสถานะปั๊มน้ำ 5 ตัว
  - แสดงกราฟิกถังน้ำ 3 ถังแบบย่อ
  - แสดงค่า Sensor หลัก (pH, EC, DO)
- **`TankLevelsPage.tsx`**: **หน้าระดับน้ำ (ละเอียด)**
  - แสดงถังน้ำแบบละเอียดทั้ง 4 ส่วน (Raw, Prep, Dist, Cultivation)
  - แสดงสถานะการไหล (Flow) และการพักน้ำ
  - ปุ่มสั่งงานปั๊มรายตัว (Manual Control)
- **`DeviceMonitorPage.tsx`**: **หน้าตรวจสอบอุปกรณ์**
  - ตารางรายชื่อ Sensor และ Actuator ทั้งหมด
  - สถานะ Online/Offline ของแต่ละตัว
- **`CropReportsPage.tsx`**: หน้ารายงานผลผลิต (แสดงตารางและกราฟสรุปการปลูก)
- **`FarmSettingsPage.tsx`**: หน้าตั้งค่าฟาร์ม (ตั้งค่าเกณฑ์ Alert, ตั้งเวลาทำงาน)
- **`WeatherDataPage.tsx`**: หน้าพยากรณ์อากาศและสภาพแวดล้อม
- **`MachinePerformancePage.tsx`**: หน้าแสดงประสิทธิภาพเครื่องจักร (OEE, Uptime)
- **`WolffiaAnalyticsPage.tsx`**: หน้าวิเคราะห์การเจริญเติบโตของไข่น้ำ (Wolffia)
- **`MaintenancePage.tsx`**: หน้าแจ้งเตือนการซ่อมบำรุง
- **`MyProfilePage.tsx`**: หน้าข้อมูลผู้ใช้งาน

## 5. UI Components (ใน folder `ui`)

โฟลเดอร์ `src/components/ui/` เก็บชิ้นส่วนเล็กๆ ที่ใช้ซ้ำกัน (สร้างจาก shadcn/ui) เช่น:

- **`button.tsx`**: ปุ่มกด
- **`card.tsx`**: กรอบสี่เหลี่ยมใส่เนื้อหา (ใช้เยอะมาก)
- **`progress.tsx`**: หลอดแสดงเปอร์เซ็นต์ (ใช้ในระดับน้ำ)
- **`sheet.tsx`**: แถบเมนูเลื่อนด้านข้าง (ใช้ในเมนูมือถือ)
- **`badge.tsx`**: ป้ายสถานะเล็กๆ (เช่น Online/Offline)
- **`dialog.tsx`**: หน้าต่าง Pop-up (ใช้ตอนกดดูรายละเอียด)
